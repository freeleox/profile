- v4.1.0: 修复输入内容无法被 el-select 组件识别的问题
  - **根因**：setNativeValue 只用原生 setter + Event('input') 设置值，Element UI 的 el-select 不能感知
  - **修复1** - 重写 setNativeValue()：
    - 首选 `document.execCommand('insertText')` 模拟真实键盘输入（最接近用户操作）
    - 回退使用 `InputEvent`（带 data/inputType 属性）替代普通 `Event`
    - 补充完整 composition 事件序列：compositionstart → compositionupdate → compositionend
    - 增加详细日志输出（execCommand 结果、最终 input.value）
  - **修复2** - 重写输入流程（第五步/第六步）：
    - 方式A：先尝试一次性输入全部 ID（空格分隔）+ 回车，依赖已勾选的空格分隔符
    - 标签验证：回车后检查 `.el-tag` 数量是否 >= 期望 ID 数
    - 方式B回退：如方式A标签不足，逐个 ID 单独输入+回车（focus→click→setNativeValue→pressEnter）
    - 最终验证：输出所有标签内容，标签为 0 时显示警告 Toast 并延迟 3 秒再保存
- v4.2.0: 修复 iframe 中重复显示导航按钮的问题
  - **问题**：脚本 `@match *://*/*` 匹配所有页面，包括 iframe 内嵌页面，导致导航按钮在登录框等 iframe 中重复出现
  - **修复**：在脚本入口处添加 `window.self !== window.top` 检测，iframe 环境下直接 return 跳过所有 UI 注入
  - 使用 try-catch 包裹检测逻辑，避免跨域 iframe 中访问 `window.top` 抛出 SecurityError
